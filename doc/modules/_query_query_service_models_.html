GetJobResultsOpts

GetJobResultsOpts: { maxWait?: undefined | number; offset?: undefined | number; pageCursor?: string | null; pageNumber?: undefined | number; pageSize?: undefined | number; resultFormat?: ResultFormat }
  • Defined in src/query/query_service_models.ts:308

Configuration options for the Get Job Results Query API call

Type declaration

  • Optional maxWait?: undefined | number

    Maximum number of milliseconds you want the HTTP connection to remain open waiting for a response. If the requested page cannot be returned in this amount of time, the service closes the connection without returning results. This parameter’s maximum value is 2000 (2 seconds). If this parameter is 0, the HTTP connection is closed immediately upon completion of the HTTP request. Default value : 0

  • Optional offset?: undefined | number

    Log record number that you want to start this page with, counting from the first log record in the result set. It is an error to use this parameter with the pageCursor and/or pageNumber parameters

  • Optional pageCursor?: string | null

    Cursor value to use for fetching this page. Each call to this API contains a response object page.pageCursor field. The value of this field is the cursor value that you use to retrieve the next page.It is an error to use this parameter with the pageNumber and/or offset parameters.

  • Optional pageNumber?: undefined | number

    Page number to fetch. Page numbers are calculated by using the default page size (10000). If this parameter is used, then pageSize is ignored. It is an error to use this parameter with the pageCursor and/or offset parameters

  • Optional pageSize?: undefined | number

    Number of log records you want retrieved for this request. The value you specify here identifies the number of records that will appear in the response object’s page.result.result array. If you are retrieving the last page in the job’s result set, this is the maximum number of records that will appear in the page.result.result array.This parameter’s maximum value is 100000. Default value : 10000

  • Optional resultFormat?: ResultFormat

    Format of the retrieved log records. Log records are returned in the response object’s page.result.result array. Each element of this array is a single log record. This parameter identifies each such record’s format. - valuesArray: Individual records are provided as an array. Each element of this log record array is a log field value. Position is meaningful in this array. Use the result object’s schema.fields array to identify the log field to which any given member of the log record array belongs. - valuesDictionary Individual log records are provided as a dictionary, or JSON object. Each such object’s field name is identical to the log record field name.

  • Defined in src/query/query_service_models.ts:143

Detailed information about a query job

Type declaration

  • Optional endTime?: undefined | number

    Ending time range for this query. Log records newer than this timestamp are not considered for query evaluation.

  • jobId: string

    The unique ID assigned to this query job.

  • Optional params?: QueryParams
  • Optional progress?: undefined | { completionPct: number }
  • Optional startTime?: undefined | number

    Starting time range for this query. Log records older than this timestamp are not considered for query evaluation. *

  • state: JobState

    Job state. Job is not completed unless the state is Done or Failed. If Pending, no job results are available.

    • pending: Job has been submitted but is not yet processing data.
    • running: Job is actively processing data. Some query pages might be available for retrieval.
    • done: Job is complete. All job data is ready for retrieval.
    • failed: Job did not successfully complete.
    • timedout: Job did not finish in requested time limit.
    • canceled: Job was terminated because of a cancel request.
  • Optional statistics?: undefined | { cachePct?: undefined | number; etaMs?: undefined | number; runTimeMs?: undefined | number }

    Provides runtime statistics such as resources used, cache hits, data volume processed, and so forth.

  • submitTime: number

    Timestamp when the query job was submitted to Cortex.
